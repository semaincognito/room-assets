# REFLECTION

1) Что оказалось самым сложным?
Самым сложным было корректно реализовать проверку пересечений бронирований по времени и сделать удобный UI для ввода времени (start/end) с валидациями.

2) Какие допущения сделаны?
Backend отсутствует — данные хранятся в localStorage. Редактирование сущностей можно добавить позже, сейчас реализованы базовые операции (создание/удаление) и обязательный минимум (поиск/фильтры, импорт/экспорт, предотвращение пересечений).

3) Что требует доработки?
- Добавить редактирование Rooms/Assets/Bookings (update)
- Привязка Assets к Rooms (например, оборудование аудитории)
- Добавить сортировку, пагинацию и расширенные фильтры
- Подключить настоящий backend и авторизацию

4) План дальнейшего развития
- Перенести storage-сервисы на backend API (Express/Nest)
- Деплой: фронтенд на GitHub Pages, backend на Render
- Настроить CI/CD через GitHub Actions
- Добавить автоматические тесты (unit/e2e)
