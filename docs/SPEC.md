# SPEC — Room Assets (Task 1)

## Цель
Сделать приложение для учета аудиторий (Rooms), ресурсов (Assets) и бронирований (Bookings) с проверкой конфликтов и возможностью импорта/экспорта данных в JSON.

## Функциональность

### Rooms
- Просмотр списка аудиторий
- Добавление аудитории (номер, название, вместимость)
- Удаление аудитории
- Валидации:
  - номер и название обязательны
  - вместимость > 0
  - номер аудитории уникален

### Assets
- Просмотр списка ресурсов
- Добавление ресурса (название, описание опционально)
- Удаление ресурса
- Валидации:
  - название обязательно
  - название ресурса уникально (без учета регистра)

### Bookings
- Просмотр бронирований
- Создание брони:
  - выбор аудитории
  - название брони
  - время начала и окончания
- Удаление брони
- Валидации:
  - все поля обязательны
  - end > start
  - запрет пересечений времени для одной аудитории

### Data (Import/Export)
- Экспорт: скачать JSON со всеми данными { rooms, assets, bookings }
- Импорт: загрузить JSON в указанном формате и заменить текущие данные

## Ограничения
- Backend не используется (данные хранятся локально).
- Технологии: React + TypeScript + MUI, хранение в localStorage.
